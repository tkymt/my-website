Androidアプリ開発 Kotlinで全画面表示する方法

以下のコードを任意のタイミングで呼び出せば全画面表示にできる

val insetsController = WindowCompat.getInsetsController(window, window.decorView)
insetsController.hide(WindowInsetsCompat.Type.systemBars())
insetsController.systemBarsBehavior = WindowInsetsControllerCompat.BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE

全文は下記の通り

MainActivity.kt 
package com.example.hidesystembars

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import androidx.core.view.WindowCompat
import androidx.core.view.WindowInsetsCompat
import androidx.core.view.WindowInsetsControllerCompat

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        hideSystemBarsForImmersiveMode()
    }

    private fun hideSystemBarsForImmersiveMode() {
        val insetsController = WindowCompat.getInsetsController(window, window.decorView)
        insetsController.hide(WindowInsetsCompat.Type.systemBars())
        insetsController.systemBarsBehavior = WindowInsetsControllerCompat.BEHAVIOR_SHOW_TRANSIENT_BARS_BY_SWIPE
    }
}

この公式ドキュメントのコードは、適切ではない。なぜならandroid.wiewとandroidx.core.viewのコードが混在しているからだ。
必ず、Compatが付いているandrodix.core.viewのクラスとメソッドを使うように。
そうすることによって全てのAPIレベルで互換性が保たれる。もちろん特定のAPIレベルを対象にした下位互換性のないコードも書くことができる。また、廃止されたAPIもある。
